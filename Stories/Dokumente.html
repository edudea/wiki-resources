Ausgangslage

Die Dokumentenverwaltung mit dem Dokumentenaustausch nimmt eine zentrale Stellung innerhalb meiner IT-World ein. Bisher nutze ich dazu die Google-Drive, leguli oder bei Kundenprojekten auch die Microsoft Dienste wie Office365 und Sharepoint etc.

Die zentrale Stellung ist damit begründet, dass die Dokumentenverwaltung auch den Datenaustausch im Team und mit Dritten sowie die gemeinsame Bearbeitung von Dokumenten ermöglichen ermöglichen muss. Die gemeinsame Bearbeitung von Dokumenten stellt hohe Anforderungen an die Integration von Datenaustauschfunktionen in die Anwendungsprogramme bei gleichzeitiger Datensicherheit. Dadurch kann das Thema der Datenverwaltung und des Datenaustauschs nicht von deer Auswahl der Apps getrennt werden. Das gewählte Vorgehen ist deshalb folgendes:

  • Es wird anhand der Anforderungen zur Datenverwaltung und zum Datenaustausch zusammen mit den Kollaborationsmöglichkeiten ein geeignetes System ausgewählt - dieses System nenne ich im weiteren Basissystem
  • Für die damit nicht abgedeckten Anwendungsgebiete werden dann weitere Systeme einzeln ausgewählt, so dass damit die Datenverwaltung und der Datenaustausch gewährleistet ist
  • Falls erforderlich muss dazu ein separates System für die Verschlüsselung der Daten eingesetzt werden - dieses System nenne ich im weiteren Verschlüsselungssystem

Auf diese Weise wird dann meine IT-Word zusammengestellt.

Anforderungen

Neben den allgemeinen Anforderungen aus MY IT World sind folgende Anforderungen vom Basissystem zu erfüllen:

  • Anbieter und Betreiber der Software hat nie Zugriff auf die Daten (Verschlüsselung der Daten mit meinem Schlüssel, Schlüssel nie beim Anbieter oder Betreiber)
  • Verwaltung von beliebigen Dateien, die von anderen Systemen erstellt werden
  • Synchronisierung von Laufwerken oder Ordnern mit Unterordnern
  • Möglichst Abdeckung vieler Anwendungsgebiete und gute Erfüllung für abgedeckte Anwendungsgebiete

Betrachtete Systeme

Die bisher verwendeten System von Google, Microsoft werden nicht weiter betrachtet, da sie dem US Cloud Act unterliegen und auch für die Verschlüsselung keine praktikablen Lösungen anbieten. In Verbindung mit einem System, das alle Daten lokal verschlüsselt und dann die verschlüsselten Daten mit einem Cloud-Dienst synchronisiert, können solche Dienste zumindest aus der Sicht des Datenschutzes verwendet werden - dies wird jedoch hier nicht weiter betrachtet. Diese Dienste können z.B. eine Alternative zu Nextcloud sein.

Die Systeme, die einen ausreichenden Datenschutz bieten, können heute immer nur ein Teil der Funktionalität abbilden, bzw. es ist eine Kombination von mehreren Systemen erforderlich, um die gewünschte Funktionalität zu erreichen. Deshalb werden nachfolgend zunächst die Einzelanbieter ausgewählt und anschliessend jeweils eine mögliche Kombination von Einzelanbietern, um die Gesamt Funktionalität abzubilden.

Für den Datenaustausch mit Dritten, d.h. mit Teilnehmern die nicht im Team sind, wird ein Messenger verwendet, sofern diese Funktion nicht durch die ausgewählten Systeme abgedeckt ist.

Folgende Dienste erfüllen die wesentlichen allgemeinen Anforderungen - erwähnt werden die allgemeinen Anforderungen, die nicht erfüllt werden:

  • boxcryptor: Boxcryptor bietet eine lokale Verschlüsselung von Dateien an, die dann mit einem gängigen Cloud Anbieter synchronisiert werden können. Boxcryptor ist nicht als Open Source verfügbar und muss auf jedem Client installiert werden (nicht browserbasiert).
  • cryptomator: Cryptomator bietet eine lokale Verschlüsselung von Dateien an, die dann mit einem gängigen Cloud Anbieter synchronisiert werden können. Cryptomator muss auf jedem Client installiert werden (nicht browserbasiert).
  • cryptpad: Cryptpad bietet einige Cloud Dienste für Zusammenarbeit im Team und Dienste für die Speicherung und Synchronisierung von allgemeinen Daten an.
  • Das Synchronisieren von Ordnern ist nicht möglich.
  • mega: Mega bietet Cloud Dienste für die Speicherung und Synchronisierung von allgemeinen Daten und für die Chat-Kommunikation an. Mega muss auf jedem Client installiert werden (nicht browserbasiert), es gibt eine Browsererweiterung für Chrome und Edge. Es ist nur die Client Software ist als Open Source verfügbar.
    Mit mega ist kein Verschlüsselungstool wie boxcryptor oder cryptomator, da alle Daten schon verschlüsselt (Zero Knowledge) in der Cloud abgelegt werden.
    . Das Synchronisieren von Ordnern ist möglich.
  • nextcloud: Nextcloud bietet umfangreiche Cloud Dienste für Zusammenarbeit im Team und Dienste für die Speicherung und Synchronisierung von allgemeinen Daten an. Dabei gibt es die Möglichkeit für die Verschlüsselung der Daten in der Cloud, wobei der Schlüssel jedoch dem Cloud-Provider bekannt ist. Nextcloud muss auf jedem Client installiert werden (nicht browserbasiert).
    . Das Synchronisieren von Ordnern ist möglich?
  • signal: Signal ist ein Messenger, der auch für den Austausch von Dateien genutzt werden kann. Signal muss auf jedem Client installiert werden  (nicht browserbasiert). Signal ist eine gemeinnützige amerikanische Stiftung. Signal unterliegt dem US Cloud Act, kann durch aufgrund dem Zero Knowledge nicht auf die Daten zugreifen.
    Signal ist kostenlos und es wird zugesichert, dass die Daten nie für Werbezwecke verwendet werden. Teile oder der gesamte Dienst kann jederzeit abgestellt werden.
  • threema: Threema ist ein Messenger, der auch für den Austausch von Dateien genutzt werden kann.
  • tresorit: Tresorit bietet Cloud Dienste für die Speicherung und Synchronisierung von allgemeinen Daten an. Tresorit ist nicht als Open Source verfügbar. Mit tresorit ist kein Verschlüsselungstool wie boxcryptor oder cryptomator, da alle Daten schon verschlüsselt (Zero Knowledge) in der Cloud abgelegt werden.
    Das Synchronisieren von Ordnern ist möglich.

Erfüllung der allgemeinen Anforderungen

Als Grundsystem wird einer der Cloud-Dienste (cryptpad, mega, nextcloud oder tresorit) verwendet, dazu ein Verschlüsselungsdienst (boxcryptor oder cryptomator) und ein Messenger (signal oder threema). Gegenüber den allgemeinen Anforderungen ergibt sich damit für die so zusammengestellten Systeme folgendes Bild:

Installation der Systeme

Alle Systeme werden so installiert, dass sie auf meinen Ubuntu Notebooks (einer privat, einer für die Firma) und auf meinem Smartphone unter Android zur Verfügung stehen. Die für die Firma erforderlichen Applikationen ebenfalls auf meinem bestehenden Window Notebook.

boxcryptor

Boxcryptor ist nicht in Ubuntu Software enthalten.

 

 

cryptomator

Cryptomator ist nicht in Ubuntu Software enthalten.

Die Installation erfolgt über die Homepage von Cryptomator: Feld “jetzt herunterladen” anklicken. Je nach Betriebssystem kommt dann die Seite zum herunterladen. Bei Ubuntu kommt rechts der Bereich “Cryptomator für Linux”. Dort bei Paketquellen “PPA: sebastian-stenzel/cryptomator” anwählen. Dort die beiden Terminal-Befehle unter “Adding this PPA to your system” per copy holen und im Terminal mit paste eintragen und ausführen. Bestätigungen geben. Wenn das fehlerlos läuft mit “sudo apt install cryptomator” installieren. Wenn das fehlerlos durchläuft, kann cryptomator z.B. über dei Aktivitäten in die Favoritenleiste übertragen und aufgerufen werden.

 

cryptpad

Nicht in Ubuntu Software enthalten.

 

mega

Nicht in Ubuntu Software enthalten.

 

nextcloud

Unter Ubuntu Software ist der nextcloud Desktop client vorhanden - auf dem Linux Notebook wird er durch anwählen und anklicken von “installieren” in Ubuntu Software installiert.

Für meine Firma in der Schweiz habe ich bei equipe zunächst mal die 14 tägige Gratisversion installiert.

signal

Signal ist nicht in Ubuntu Software enthalten.

Signal muss zunächst auf dem Smartphone, meines mit Android, installiert werden. Dazu wird die App aus dem Sore heruntergeladen und installiert, dann muss die Mobilnummer und eine PIN angegeben werden und der Benutzername wird abgefragt. Die PIN wird für die Verschlüsselung verwendet und muss sicher aufbewahrt werden, bei mir im 1password. Die Kontakte werden verknüpft - da diese nie durch Signal im Klartext lesbar sind ist das unproblematisch ist. Nach der der Rückfrage auf dem Handy ist die App installiert und einfach bedienbar.

Für die Installation unter Linux gibt es eine Anleitung bei Signal. Diese ist jedoch nicht korrekt, da veraltete Verzeihnisnamen verwendet werden. Nachdem das schon erfolglos aufgerufen bae, musste ich in Ubuntu mit der Konsole die schon übertragenen Dateien löschen. Dann habe ich die Anleitung unter Ubunterusers zur Installation des Signal Desktops verwendet. Das funktioniert einwandfrei.

Nach der Installation wird Signal über die Aktivitäten mit der Favoritenleiste verknüpft und dann aufgerufen.
